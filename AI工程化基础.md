#### 模型能力增强

常见的通用的知识（基座模型）来源大部分是互联网上的资料（公域知识）
私域知识：不在互联网上，无法被检索，自然不能被基模型训练进去

增强模型两大类：

1. 从模型的本身去增强（成本高，需要模型团队长达数周数月的训练增强评测，并且还会有「微调税」，也就是灾难性遗忘，就是已经训练好的基座模型已经掌握的知识，在微调后会忘记）
   1.1 专业的算法团队做增强
   1.2 再训练
   1.2.1 微调（在基座模型的基础上，使用少量数据进行训练，使模型适应特定任务）
   1.2.2 模型蒸馏（大模型在特定任务上提取出一个较小的模型）
   1.2.3 剪枝和拼接 - 剪少模型的参数规格和模型层数

2. AI 工程的增强
   2.1 Prompt
   2.2 Function Calling(tools)：Chat bot 应用的 deepseek，早期豆包，chatGPT 问答形式。在模型本身能力的基础上，增加一些工具调用能力。
   函数注册 => 输入文本，由模型决定是否调用（意图识别）

Function Call 实现原理（后来叫 tools）：

1.  模型侧：通过训练和模型能力增强，让模型可以输出特定规格的文本结构
2.  工程测：处理模型返回，提取需要执行的函数信息，并最终代替模型执行

Function Call 和 tools 的区别：

1. tools 支持并行，function call 是串行
2. tools 代表 llm agent 工具（不仅限于函数调用），function 只代表函数调用
3. 跨执行器环境的能力
